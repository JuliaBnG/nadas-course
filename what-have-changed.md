# What have changed after adding packages

After adding the packages, `Project.toml` will look like this:

```toml
name = "flfSheep"
uuid = "866d409a-f53d-443e-8628-6d86ebd581db"
version = "0.1.0"
authors = ["Xijiang Yu <xijiang@users.noreply.github.com>"]

[deps]
BnGStructs = "764f5799-4fa8-443f-b488-b719b6ff37e3"
FisherWright = "a555c6d9-8fd4-4ecd-87f6-904b8d8d9cbe"
RelationshipMatrices = "d4d45922-7dca-4708-ac5a-a6dbb0551829"

[compat]
BnGStructs = "0.2.2"
FisherWright = "0.1.6"
RelationshipMatrices = "0.1.4"
```

Note if you want to register your package in the Julia General Registry, some
tweaks are needed.  That is beyond the scope of this course.  With the help of
AI, e.g., `GitHub Copilot`, that is not a big deal.

There is also a new file `Manifest.toml` in the project directory.

```bash
echo Manifest.toml >> .gitignore
git add .gitignore
```

Above means that we will not track `Manifest.toml` in `git`.  The reason is that
`Manifest.toml` is automatically generated by `Julia`.  It can change very much
from time to time. It is thus excluded from version control.
