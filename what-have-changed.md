# Changes After Adding Packages

Once the packages are added, `Project.toml` will be updated to include the 
dependencies and compatibility requirements:

```toml
name = "flfSheep"
uuid = "866d409a-f53d-443e-8628-6d86ebd581db"
version = "0.1.0"
authors = ["Xijiang Yu <xijiang@users.noreply.github.com>"]

[deps]
BnGStructs = "764f5799-4fa8-443f-b488-b719b6ff37e3"
Breeding = "cb565f4f-111b-40c2-8768-3bafadb18772"
DataFrames = "a93c6f00-e57d-5684-b7b6-d8193f3e46c0"
FisherWright = "a555c6d9-8fd4-4ecd-87f6-904b8d8d9cbe"
Random = "9a3f8284-a2c9-5f02-9a11-845980a1fd5c"
RelationshipMatrices = "d4d45922-7dca-4708-ac5a-a6dbb0551829"

[compat]
BnGStructs = "0.2.2"
Breeding = "0.1.0"
DataFrames = "1.8.1"
FisherWright = "0.1.6"
Random = "1.11.0"
RelationshipMatrices = "0.1.4"
```

Note that if you intend to register your package in the Julia General 
Registry, additional configuration will be required. This is beyond 
the scope of this course, but can be easily managed with the 
assistance of AI tools like GitHub Copilot.

A new `Manifest.toml` file will also be created in the project directory.

```bash
echo Manifest.toml >> .gitignore
git add .gitignore
```

The command above ensures that `Manifest.toml` is not tracked by Git. 
Since `Manifest.toml` is automatically generated by Julia and can change 
frequently based on the local environment, it is typically excluded 
from version control.
